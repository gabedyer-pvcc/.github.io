    Operating systems Terminology
    Term Definition
    Operating System An operating system is the software that controls the overall operation of a computer.
    Job Queue an ordered list of jobs waiting to be processed by a subsystem
    FIFO A method of organizing, handling, and manipulating the data structure of elements in a computing system
    Interactive processing The process of interacting directly with a program and/or data for immediately modifying, retrieving and/or displaying information
    Time-Sharing Method of operation in which multiple users with different programs interact nearly simultaneously with the central processing unit (CPU) of a large-scale digital computer
    Multiprogramming The running of two or more programs or sequences of instructions simultaneously by a computer with more than one central processor
    Application software A computer program designed to carry out a specific task other than one relating to the operation of the computer itself, typically to be used by end-users
    System software A type of computer program that is designed to run a computer's hardware and application programs
    Utility software A program specifically designed to help manage and tune system or application software
    Graphical user interface (GUI) A form of user interface that allows users to interact with electronic devices through graphical icons and audio indicators
    Shell Is the layer of programming that understands and executes the commands a user enters
    Kernel A core component of an operating system and serves as the main interface between the computer's physical hardware and the processes running on it
    Device driver A special kind of software program that controls a specific hardware device attached to a computer
    File manager A computer program that provides a user interface to manage files and folders
    Memory manager The process of controlling and coordinating a computer's main memory
    Vitural memory Uses both hardware and software to enable a computer to compensate for physical memory shortages, temporarily transferring data from random access memory (RAM) to disk storage
    Boot strapping To boot or to load a program into a computer using a much smaller initial program to load in the desired program
    ROM a non-volatile memory type. This means it receives data and permanently writes it on a chip, and it lasts even after you turn off your computer. The data is coded to not be overwritten
    Deadlock a situation in which two computer programs sharing the same resource are effectively preventing each other from accessing the resource, resulting in both programs ceasing to function
    Spooling a process in which data is temporarily held to be used and executed by a device, program or the system. Data is sent to and stored in memory or other volatile storage until the program or computer requests it for execution
    Super user (or administrator) someone who can make changes on a computer that will affect other users of the computer. Administrators can change security settings, install software and hardware, access all files on the computer, and make changes to other user accounts.
    Multi-factor authentication a multi-step account login process that requires users to enter more information than just a password
