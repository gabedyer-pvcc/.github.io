    All About Github?

    Q.1 What is a version control System?

    A.1 tracks the history of changes as people and teams collaborate on projects together.

    Q.2 What is Git?

    A.2 Git lets developers see the entire timeline of their changes, decisions, and progression of any project in one place. From the moment they access the history of a project, the developer has all the context they need to understand it and start contributing.

    Q.3 What is Github?

    A.3 GitHub hosts Git repositories and provides developers with tools to ship better code through command line features, issues (threaded discussions), pull requests, code review.

    Q.4 What is the diffrence between Git and Github?

    A.4 Git manage and keep track of your source code history GitHub is a cloud-based hosting service that lets you manage Git repositories.

    Q.5 Who Started Github and how was it started?

    A.5 Tom Preston-Werner, Chris Wanstrath, and PJ Hyett, GitHub was created to provide developers the option to host code securely and manage commits to code in proper manner.

    Q.6 What company owns It now?

    A.6 Microsoft Corporation.

    Github's Mascot

    Git/Github terms :

  • Repository- A repository contains all of your project's files and each file's revision history. You can discuss and manage your project's work within the repository.
  • Commit- Save
  • Fork- A fork is a new repository that shares code and visibility settings with the original “upstream” repository.
  • Push- You can use secret scanning to prevent supported secrets from being pushed into your organization or repository by enabling push protection. (security)
  • Pull request- Pull requests let you tell others about changes you've pushed to a branch in a repository on GitHub. Once a pull request is opened, you can discuss and review the potential changes with collaborators and add follow-up commits before your changes are merged into the base branch.
  • Workflows- A workflow is a configurable automated process that will run one or more jobs
  • Issues- Use GitHub Issues to track ideas, feedback, tasks, or bugs for work on GitHub.
  • Raw button- the raw URL for a file
  • Blame button- is used to know who/which commit is responsible for the latest changes made to a file.
